// 03. 实战流程

5 步让 AI 帮你写代码

这是一套经过验证的实战流程，适用于任何 AI 编程工具。 掌握这 5 步，你就能高效地与 AI 协作。

01

明确需求

清晰描述你想要实现什么。越具体，AI 的输出越精准。

示例 
❌ 模糊的描述：
"帮我写个网络程序"

✓ 清晰的描述：
"用 C 语言实现一个高性能 TCP 服务器：
- 使用 epoll 实现多路复用
- 支持至少 10000 并发连接
- 每个连接需要独立的读缓冲区
- 目标平台是 Linux (CentOS 7)"
💡 技巧：说明语言、平台、性能要求、约束条件

根据任务类型选择合适的 AI 工具。

告诉 AI 你的项目是如何组织的

04

生成代码

AI 生成代码后，你需要审查和验证。

✓ 检查逻辑是否正确
✓ 验证边界条件处理
✓ 确认内存安全（C 语言）
✓ 检查错误处理完整性
✓ 验证是否符合编码规范
⚠️
重要提醒

AI 生成的代码不是 100% 正确的。作为资深开发者，你的审查是必不可少的环节。

05

调试优化

遇到问题？让 AI 帮你调试。迭代优化直到满意。

调试对话示例 
你: 代码编译报错了，错误信息是：
    "undefined reference to 'epoll_create1'"

AI: 这个错误说明你的系统 epoll 支持可能有问题。
    请检查：
    1. 内核版本 (需要 2.6.27+)
    2. 编译时是否正确链接

    尝试在编译命令中添加 -lrt 参数：
    gcc -o server server.c -lrt

你: 加了 -lrt 还是报错

AI: 让我看看你的代码... 问题出在头文件。
    epoll_create1 需要 _GNU_SOURCE 宏。

    在文件开头添加：
    #define _GNU_SOURCE
    #include <sys/epoll.h>
提问 → 获得解答 → 尝试 → 反馈结果 → 继续优化

场景描述

华为云 ECS 实例运行高负载计算任务，需要优化内核调度参数以减少延迟抖动，提升实时性能。

提示词模板

请帮我优化华为云 ECS 的内核调度参数：

环境信息：
- 华为云 ECS（鲲鹏 920 / x86 可选）
- OS：openEuler 22.03 / CentOS 7.9
- 内核：5.10+ / 4.19 LTS
- 工作负载：实时数据处理的计算密集型任务

当前问题：
- CPU 调度延迟抖动较大（p99 > 500us）
- 多核负载不均衡
- 中断处理占用过多 CPU 时间

优化目标：
- 降低调度延迟和抖动
- 提高 CPU 缓存命中率
- 优化 NUMA 亲和性
- 隔离关键业务 CPU

请提供：
1. 内核启动参数调优建议（isolcpus、nohz_full 等）
2. sysctl 调度器参数配置
3. IRQ 亲和性配置脚本
4. cgroups v2 资源隔离方案
5. 验证和监控方法

常用调优参数

  • isolcpus=1-3 隔离 CPU 核心
  • nohz_full=1-3 禁用时钟中断
  • rcu_nocbs=1-3 RCU 回调卸载
  • kernel.sched_min_granularity_ns 最小调度粒度
  • kernel.sched_wakeup_granularity_ns 唤醒粒度

场景描述

华为云上的 KVM 虚拟机网络 I/O 性能不达标，需要调优 virtio 和后端驱动。

提示词模板

请帮我优化 KVM 虚拟机的网络 I/O 性能：

环境信息：
- 宿主机：华为云物理机，openEuler 22.03
- 虚拟机：CentOS 7.9，virtio-net 网卡
- 业务：高吞吐量网络转发（10Gbps+）
- CPU：支持 DPDK 和 vhost-user

当前问题：
- 网络吞吐量只能达到 6Gbps
- CPU 在软中断上消耗过高（30%+）
- P99 延迟不稳定

请提供优化方案：
1. virtio-net 参数调优（多队列、合并缓冲区）
2. vhost-net vs vhost-user 选择建议
3. CPU 亲和性绑定（vCPU 和 vhost 线程）
4. 内存大页配置（HugePages）
5. 中断合并和 RSS 配置
6. 性能测试和验证方法

关键调优点

  • virtio-net 多队列：MQ=on, vectors=多队列数*2+2
  • vhost 后端线程 CPU 亲和性绑定
  • 启用 RSS 和中断合并减少 CPU 开销
  • 大页内存减少 TLB miss
